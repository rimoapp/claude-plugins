# claude-plugin-auto-worktree

[English](../../README.md) | [日本語](README.ja.md) | [Español](README.es.md) | [Deutsch](README.de.md) | [中文](README.zh-cn.md) | [Русский](README.ru.md) | [Português](README.pt.md) | [한국어](README.ko.md)

Плагин для Claude Code, который автоматически перенаправляет Claude в git worktree перед изменением файлов, обеспечивая безопасную параллельную работу без конфликтов в git.

## Проблема

Когда несколько сессий Claude Code одновременно работают с одним репозиторием, изменения файлов могут конфликтовать. Пользователи, не знакомые с ветвлением в git, могут потерять свою работу или столкнуться с запутанными конфликтами слияния.

## Принципы проектирования

**При обычном использовании изменения кода происходят в ветках worktree.** Это руководящий принцип, а не жёсткое ограничение для каждой команды.

Плагин спроектирован с минимальным вмешательством:

- **`Write`/`Edit` для отслеживаемых файлов** в основном репозитории блокируются — Claude перенаправляется на создание worktree
- **Команды `Bash`** практически полностью разрешены — блокируются только перенаправления вывода (`>`, `>>`) в отслеживаемые файлы репозитория
- **Команды Git** (`checkout`, `reset`, `merge`, `rebase`, `stash` и т.д.) всегда разрешены — текущая основная ветка не считается заведомо корректной, и пользователям может потребоваться её исправить или управлять ею
- **Менеджеры пакетов, системные команды, файловые утилиты** — всё разрешено
- **Запись в `/tmp`, игнорируемые git пути или файлы вне репозитория** всегда разрешена (Plan Mode, память, временные файлы — всё работает)

## Решение

Плагин перехватывает вызовы инструментов `Write`, `Edit` и `Bash` через хук `PreToolUse`. Когда Claude пытается записать или отредактировать отслеживаемый файл в основном репозитории, плагин:

1. Блокирует изменение (код выхода 2)
2. Инструктирует Claude вызвать встроенный инструмент `EnterWorktree`
3. Claude создаёт изолированный worktree и повторяет действие там

Каждая сессия Claude получает собственный изолированный worktree и ветку, поэтому параллельные сессии никогда не конфликтуют.

## Установка

### Из GitHub (рекомендуется)

В Claude Code выполните:

```
/plugin marketplace add rimoapp/claude-plugin-auto-worktree
/plugin install auto-worktree@rimoapp-plugins
```

После установки плагин сохраняется между сессиями. Вы можете включать и отключать его в любое время:

```
/plugin disable auto-worktree@rimoapp-plugins
/plugin enable auto-worktree@rimoapp-plugins
```

### Из локальной директории

Для разработки или тестирования:

```bash
claude --plugin-dir /path/to/claude-plugin-auto-worktree
```

## Как это работает

```
Пользователь запускает Claude в основном репозитории
         │
         ▼
Срабатывает хук SessionStart ─── На ветке по умолчанию? → Проактивно инструктирует Claude использовать EnterWorktree
         │
         ▼
Claude вызывает EnterWorktree → создаёт .claude/worktrees/<name>/
         │
         ▼
Все изменения файлов безопасно происходят в worktree
         │
         ▼
Сессия завершается → Хук Stop выводит сводку (ветка, незакоммиченные изменения)
```

Если Claude пропускает проактивную инструкцию, **хук PreToolUse** действует как страховочная сеть:

```
Claude пытается записать/отредактировать файл на ветке по умолчанию
         │
         ▼
Хук PreToolUse перехватывает ──────── Уже в worktree? → Разрешить
         │
         ▼
Блокирует действие (exit 2) + инструктирует Claude вызвать EnterWorktree
```

### Расположение Worktree

Worktree создаются встроенным инструментом `EnterWorktree` Claude Code внутри репозитория:

```
my-project/
├── .claude/
│   └── worktrees/
│       ├── humble-prancing-conway/    # Сессия 1
│       └── brave-dancing-turing/      # Сессия 2
├── src/
└── ...
```

Каждый worktree получает ветку с именем `worktree-<session-name>`.

### Фильтрация команд Bash

Плагин блокирует только команды Bash, использующие перенаправление вывода (`>`, `>>`) для записи в отслеживаемые файлы внутри репозитория. Всё остальное разрешено:

- **Разрешено**: все команды без перенаправлений (`git checkout`, `npm install`, `rm`, `touch`, `mv` и т.д.), перенаправления в `/tmp`, `/dev/null`, игнорируемые git файлы или пути вне репозитория
- **Заблокировано**: `echo "data" > tracked-file.txt`, `cat input >> src/main.py` и т.д. (перенаправления в отслеживаемые файлы репозитория)

## Конфигурация

Плагин поддерживает настраиваемые пользователем параметры через механизм `userConfig` Claude Code. После установки плагина вы можете задать эти параметры в файле `~/.claude/settings.json` в разделе `pluginConfigs`:

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `skip_directories` | Список путей к корневым директориям git-репозиториев через запятую, где auto-worktree не должен активироваться | (пусто) |
| `pull_default_branch` | Подтягивать последнюю версию ветки по умолчанию из origin при старте сессии. Используется только fast-forward — локальные изменения никогда не перезаписываются. При ошибке молча продолжает работу. | `true` |
| `sync_gitignored_writes` | Автоматически копировать игнорируемые git файлы, записанные в worktree, обратно в основной репозиторий. Охватывает вызовы инструментов Write/Edit и перенаправления вывода Bash. | `true` |

### Пример settings.json

```json
{
  "pluginConfigs": {
    "auto-worktree@rimoapp-plugins": {
      "options": {
        "skip_directories": "/Users/me/notes,/Users/me/scratch",
        "pull_default_branch": "false",
        "sync_gitignored_writes": "true"
      }
    }
  }
}
```

### skip_directories

Репозитории, корневой путь которых совпадает с записью в этом списке, будут полностью игнорироваться плагином — без принудительного использования worktree, без инструкций при старте сессии. Сопоставление основано на корневой директории git-репозитория, поэтому указание `/Users/me/notes` пропустит весь репозиторий независимо от того, в какой поддиректории работает Claude. Полезно для личных репозиториев, заметок или экспериментальных директорий, где вы хотите редактировать напрямую в ветке по умолчанию.

### pull_default_branch

Когда включено (по умолчанию), плагин выполняет `git pull --ff-only` при старте сессии (с таймаутом 8 секунд), чтобы убедиться, что локальная ветка по умолчанию актуальна перед созданием worktree. Если pull не удаётся (например, нет сети, таймаут, расхождение истории), плагин продолжает работу с локальным состоянием и выводит предупреждение. Установите значение `false`, чтобы полностью пропустить этот шаг.

### sync_gitignored_writes

Когда включено (по умолчанию), файлы, записанные по игнорируемым git путям внутри worktree, автоматически копируются обратно в основной репозиторий. Это гарантирует, что артефакты сборки в директориях вроде `dist/` или `build/` не будут потеряны при удалении worktree.

**Что синхронизируется:**
- Файлы, записанные через инструменты Write/Edit по игнорируемым git путям внутри репозитория
- Перенаправления вывода Bash (`>`, `>>`) по игнорируемым git путям внутри репозитория

**Что НЕ синхронизируется:**
- Файлы, созданные косвенно командами (например, `npm install`, создающий `node_modules/`)
- Файлы вне репозитория (например, `/tmp/...`)
- Файлы по отслеживаемым (не игнорируемым git) путям

Установите значение `false`, чтобы полностью отключить это поведение.

## Обход для сессии

Если плагин ошибочно блокирует действие, вы можете попросить Claude пропустить принудительное использование worktree для текущей сессии на естественном языке — подойдёт любая формулировка:

- "worktree作らなくていい" / "auto-worktree 無視して"
- "don't need a worktree" / "skip worktree" / "no worktree please"
- Или любой другой способ выразить то же намерение

Claude выполнит `touch <bypass-flag-file>`, чтобы отключить принудительное использование worktree до конца сессии. Флаг хранится во временной директории системы (`$TMPDIR` / `$TMP` / `$TEMP` / `/tmp`) и **не** влияет на другие сессии.

## Очистка

Очистка worktree выполняется встроенным инструментом `ExitWorktree` Claude Code. Когда сессия завершается в worktree, пользователю предлагается сохранить или удалить его.

Для ручной очистки:

```bash
git worktree list          # Просмотр всех worktree
git worktree remove <path> # Удаление конкретного worktree
git worktree prune         # Очистка устаревших ссылок
```
